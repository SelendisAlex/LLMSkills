# LLMSkills 工作架构

## 项目定位

`D:\Code\LLMSkills` 是一个用于维护 Codex Skills 和工作记忆的轻量仓库。当前仓库没有应用运行时、包管理文件、构建脚本或测试框架，主要维护对象是 Markdown 形式的 Skill 说明、全局代理规则、工作规则记录和架构记录。

## 顶层结构

- `.gitattributes`：启用 Git 文本文件自动换行规范化，当前内容为 `* text=auto`。
- `LLMSkills.code-workspace`：VS Code 工作区文件，当前仅打开仓库根目录。
- `Agents/.codex/AGENTS.md`：代理级指令，要求每轮对话始终读取并使用全局 Skills。
- `.agents/`：当前未发现文件，暂不承担维护职责。
- `Skills/`：仓库内的 Skill 副本目录，当前包含 `firstinit` 和 `checkpointsave`。
- `archsave/`：独立 Skill 目录，当前包含 `archsave/SKILL.md`。
- `workCheckpoint/`：长期工作规则和工作过程记录。
- `workArchitecture/`：架构文档目录，本文件即当前架构记录。

## Skill 目录

### `Skills/firstinit`

`Skills/firstinit/SKILL.md` 定义工作区基础规则：

- Skill 文件使用 UTF-8 读写。
- 本工作区编写 Skill 时始终使用英文。
- 用户是中文用户，默认使用中文回复。

### `Skills/checkpointsave`

`Skills/checkpointsave/SKILL.md` 定义 checkpoint 工作流：

- 重要任务开始时读取 `workCheckpoint/Rules/` 和 `workCheckpoint/Works/` 下的 Markdown 记录。
- 用户给出长期项目约束、编码规则、架构约定或维护说明时，保存为工作记录或规则。
- 工作规则和工作记录使用中文记录。
- 任务结束时如产生长期约束、重要推理、架构变化或显著文件变更，应保存 checkpoint。

### `archsave`

`archsave/SKILL.md` 定义架构记录工作流：

- 在重要任务前读取已有 `workArchitecture/` Markdown 记录。
- 当用户明确提出 `Organize the Work Architecture` 时，扫描项目文档、框架、目录结构和关键文件，整理为便于后续开发维护的技术文档。
- 架构文档保存到当前工作目录下的独立文件夹。

注意：当前工作区的 `archsave/SKILL.md` 曾加入中文条目；后续修改任何 Skill 文件时，应遵守最新工作记录中的长期约束，Skill 文件内容始终使用英文。

## 工作记忆目录

### `workCheckpoint/Rules/`

该目录保存长期规则。目前记录的核心规则包括：

- Skill 文件使用 UTF-8 编码。
- Skill 正文和 frontmatter 使用英文维护。
- 工作规则、工作记录和架构记录使用中文保存。
- 重要任务开始前读取 checkpoint 和 architecture 记录。
- 修改 Skill 触发条件时必须同步检查 frontmatter `description`。
- 仓库内 Skill 副本不会自动同步到已安装的全局 Skill 目录。

### `workCheckpoint/Works/`

该目录保存历史工作过程。重要记录包括：

- 将 `firstinit`、`checkpointsave` 翻译为英文。
- 为 `checkpointsave` 和 `archsave` 增加任务开始读取记录的触发逻辑。
- 将架构职责从 `firstinit` 迁移到 `archsave`。
- 调整 `archsave` 的 Mandatory Startup Check 位置。
- 记录“Skill 始终英文修改约束”：维护、创建、调整任何 `SKILL.md` 或 Skill 相关正文、frontmatter、说明条目时，Skill 文件内容始终使用英文。

## 维护约束

- 修改 Skill 文件时，正文和 frontmatter 必须使用英文。
- 工作记录、工作规则和架构记录继续使用中文。
- 修改 `SKILL.md` 时保持原有 frontmatter、章节结构、路径约定和行为意图，避免无关重写。
- 新增或调整触发条件时必须检查并必要时更新 frontmatter `description`，因为它是 Skill 自动触发的重要依据。
- 对 Skill 文件做结构性修改后，优先运行 `quick_validate.py` 校验目标 Skill。Windows 环境下如遇默认 GBK 解码失败，应使用 `PYTHONUTF8=1` 重新运行。
- 仓库内 Skill 副本与已安装的全局 Skill 目录不是自动同步关系；需要跨工作区生效时，应另行执行安装或同步步骤。
- 不要删除或重写非本次任务产生的用户改动；Git 工作区可能存在未提交变更。

## 验证方式

- 查看文件列表：`rg --files`
- 查看 Git 状态：`git status --short`
- 校验 Skill：`$env:PYTHONUTF8='1'; python C:\Users\Selendis\.codex\skills\.system\skill-creator\scripts\quick_validate.py .\目标Skill目录`
- 检查空白错误：`git diff --check -- <path>`
