# 工作规则整理

- 本仓库中的 Skill 文件使用 UTF-8 编码读写。
- 本仓库中的 Skill 正文和 frontmatter 使用英文维护；工作规则与工作记录使用中文记录。
- 处理代码、审查、重构、架构、调试、项目规则等重要任务时，任务开始应先读取 `workCheckpoint/Rules/` 与 `workCheckpoint/Works/` 下的 Markdown 记录，并将其作为当前工作上下文。
- `SKILL.md` 的 frontmatter `description` 是技能自动触发的重要依据；新增或调整触发条件时，必须同步检查 description，不能只修改正文。
- 修改 `Skills/` 下的仓库副本不会自动同步到已安装的全局技能目录；需要跨工作区生效时，应额外执行安装或同步步骤。
- 维护 Skill 时保留原有 frontmatter、章节结构、路径约定和行为意图，避免无关重写。
- 任务结束后如形成长期约束、重要推理、架构变化、维护注意事项或显著文件变更，应按 checkpointsave 规则保存工作记录。
